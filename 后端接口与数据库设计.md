## 一、设计原则

1. **RESTful 风格**：资源导向 URL，HTTP 动词表达操作语义。
2. **权限服务端强制**：所有写操作及敏感读操作在服务端校验 Owner / Member 角色，不依赖客户端过滤。
3. **软删除 + 硬删除分离**：工厂、款式走软删除；回收站「彻底删除」走级联硬删除；收发记录删除为硬删除。
4. **与小程序数据模型兼容**：字段命名、业务规则与原 CloudBase 集合保持一致，便于迁移。
5. **幂等与防重复**：创建类接口支持 `Idempotency-Key` 请求头（可选），服务端对同一 Key 返回相同结果。

## 二、技术选型建议

| 层级 | 推荐方案 | 说明 |
|------|----------|------|
| API 框架 | Node.js (NestJS) / Go (Gin) | 按团队栈选择 |
| 数据库 | PostgreSQL / MySQL 8 | 支持 JSON 字段存 `items`、`colors` |

| 原小程序 | 原生 App |

| 微信 OPENID | `users.id`（UUID） |
| `ownerId` / `members.openid` | `owner_id` / `factory_members.user_id` |

## 三、数据库 Models 设计

### 3.1 ER 关系概览

```
users ──┬──< factory_members >── factories ──< styles ──< records
        │                           │
        │                           ├──< settlements
        │                           └──< invitations
        └── (created_by on records)
```

### 3.2 users（用户表）

> 原生 App 需独立用户体系；微信登录用户可绑定 `wechat_openid`。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 用户唯一 ID |
| `nickname` | VARCHAR(64) | NOT NULL, DEFAULT '' | 昵称（展示用，头像取首字） |
| `phone` | VARCHAR(20) | UNIQUE, NULL | 手机号（手机号登录） |
| `wechat_openid` | VARCHAR(64) | UNIQUE, NULL | 微信开放平台 OPENID |
| `wechat_unionid` | VARCHAR(64) | UNIQUE, NULL | 微信 UnionID（可选） |
| `avatar_url` | VARCHAR(512) | NULL | 头像 URL（扩展） |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

**索引**：`phone`, `wechat_openid`

---

### 3.3 factories（上游工厂）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 工厂 ID |
| `name` | VARCHAR(100) | NOT NULL | 工厂名称 |
| `owner_id` | UUID | FK → users.id, NOT NULL | 老板（Owner） |
| `deleted` | BOOLEAN | NOT NULL, DEFAULT false | 软删除标记 |
| `deleted_at` | TIMESTAMP | NULL | 移入回收站时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

**业务规则**：
- 同一 `owner_id` 下未删除工厂名称建议唯一（可选约束，防误操作）。
- 删除工厂时软删除，同时软删除其下所有款式；收发记录保留至彻底删除。

**索引**：
- `(owner_id, deleted, created_at DESC)`
- `(deleted, owner_id)` — 回收站查询

---

### 3.4 factory_members（工厂成员）

> 将原 `members[{openid, role}]` 拆为独立表，便于查询与扩展。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 成员关系 ID |
| `factory_id` | UUID | FK → factories.id, NOT NULL | 所属工厂 |
| `user_id` | UUID | FK → users.id, NOT NULL | 成员用户 |
| `role` | ENUM('owner','member') | NOT NULL | 角色 |
| `joined_at` | TIMESTAMP | NOT NULL | 加入时间 |

**唯一约束**：`(factory_id, user_id)`

**索引**：`(user_id)` — 查用户有权限的工厂列表

**说明**：创建工厂时自动插入一条 `role=owner` 的记录。

---

### 3.5 styles（款式）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 款式 ID |
| `factory_id` | UUID | FK → factories.id, NOT NULL | 所属工厂 |
| `style_code` | VARCHAR(50) | NOT NULL | 款式编号，如 `10086` |
| `unit_price` | DECIMAL(10,2) | NOT NULL, DEFAULT 0 | 加工单价（元/件） |
| `colors` | JSON | NOT NULL, DEFAULT '[]' | 颜色名数组，如 `["红色","蓝色"]` |
| `deleted` | BOOLEAN | NOT NULL, DEFAULT false | 软删除 |
| `deleted_at` | TIMESTAMP | NULL | 移入回收站时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

**唯一约束**：`(factory_id, style_code)` WHERE `deleted = false`（部分唯一索引）

**colors JSON 示例**：
```json
["红色", "蓝色", "绿色"]
```

**业务规则**：
- 同一工厂下未删除款式编号不可重复（服务端校验）。
- 保存时至少 1 种颜色。
- 从收发页「＋颜色」新增颜色时，更新 `colors` 数组。

**索引**：
- `(factory_id, deleted, created_at DESC)`
- `(factory_id, style_code)` — 唯一性校验

---

### 3.6 records（收发记录）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 记录 ID |
| `factory_id` | UUID | FK → factories.id, NOT NULL | 冗余，加速统计查询 |
| `style_id` | UUID | FK → styles.id, NOT NULL | 所属款式 |
| `type` | ENUM('out','in') | NOT NULL | `out`=发出，`in`=收回 |
| `date` | DATE | NOT NULL | 业务日期 |
| `items` | JSON | NOT NULL, DEFAULT '{}' | 颜色→件数映射 |
| `remark` | VARCHAR(500) | NULL | 备注（仅 `type=out` 时使用） |
| `created_by` | UUID | FK → users.id, NOT NULL | 录入人 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

**items JSON 示例**：
```json
{
  "红色": 36,
  "蓝色": 24
}
```

**唯一约束**：`(style_id, type, date)` — 同一款式、同一日期、同一类型仅一条记录

**业务规则**：
- 数量为整数件数（客户端解析「3打」「1打2」后提交）。
- 更新某颜色格时 merge 到 `items`；数量为 0 或删除该 key。
- 长按删除行 = 删除该 `(style_id, type, date)` 整条记录（硬删除）。
- Member 只读，Owner 可写。

**索引**：
- `(style_id, date DESC)`
- `(factory_id, date)` — 统计聚合
- `(style_id, type, date)` UNIQUE

---

### 3.7 settlements（结算标记）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 结算 ID |
| `factory_id` | UUID | FK → factories.id, NOT NULL | 所属工厂 |
| `date_from` | DATE | NULL | 区间起始（NULL 表示不限） |
| `date_to` | DATE | NULL | 区间结束（NULL 表示不限） |
| `status` | ENUM('settled','unsettled') | NOT NULL | 已结清 / 未结 |
| `marked_by` | UUID | FK → users.id, NOT NULL | 标记人 |
| `marked_at` | TIMESTAMP | NOT NULL | 标记时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

**唯一约束**：`(factory_id, date_from, date_to)` — 同一工厂同一区间唯一

**说明**：统计页切换结算状态时 upsert 此记录。

---

### 3.8 invitations（邀请码）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 邀请 ID |
| `factory_id` | UUID | FK → factories.id, NOT NULL | 目标工厂 |
| `factory_name` | VARCHAR(100) | NOT NULL | 冗余工厂名（分享展示） |
| `inviter_id` | UUID | FK → users.id, NOT NULL | 邀请人 |
| `code` | CHAR(6) | UNIQUE, NOT NULL | 6 位大写字母数字 |
| `expire_at` | TIMESTAMP | NOT NULL | 过期时间（创建后 7 天） |
| `used_by` | UUID | FK → users.id, NULL | 使用者（可选，一次性邀请） |
| `used_at` | TIMESTAMP | NULL | 使用时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |

**索引**：`code`, `(factory_id, expire_at)`

**业务规则**：
- 仅 Owner 可创建邀请。
- 过期或已使用的邀请码不可再次使用（若设计为一次性）。

## 四、通用约定

### 4.1 Base URL

```
https://api.example.com/v1
```

### 4.2 请求头

| Header | 说明 |
|--------|------|
| `Authorization` | `Bearer <access_token>` |
| `Content-Type` | `application/json` |
| `Idempotency-Key` | 可选，UUID，防重复创建 |
| `X-Request-Id` | 可选，链路追踪 |

### 4.3 统一响应格式

**成功**：
```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

**失败**：
```json
{
  "code": 40001,
  "message": "无权操作",
  "data": null
}
```

### 4.4 分页（列表接口）

Query 参数：`page=1&pageSize=20`（默认 pageSize=20，最大 100）

```json
{
  "code": 0,
  "data": {
    "list": [],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 56,
      "hasMore": true
    }
  }
}
```

### 4.5 错误码

| code | HTTP | 说明 |
|------|------|------|
| 0 | 200 | 成功 |
| 40001 | 403 | 无权操作 |
| 40002 | 403 | 无权查看该工厂数据 |
| 40003 | 401 | 未登录或 Token 过期 |
| 40004 | 404 | 资源不存在 |
| 40005 | 409 | 款式编号重复 |
| 40006 | 409 | 颜色名称重复 |
| 40007 | 422 | 参数校验失败 |
| 40008 | 422 | 数量格式错误 |
| 40009 | 410 | 邀请码已过期 |
| 40010 | 409 | 邀请码无效或已使用 |
| 50001 | 500 | 服务器内部错误 |

### 4.6 时间格式

- 日期：`YYYY-MM-DD`
- 时间：`ISO 8601`（`2026-06-21T08:00:00+08:00`）

## 六、工厂模块

### 6.1 获取工厂列表（首页）

```
GET /factories
```

**Query**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `keyword` | string | 可选，按名称模糊搜索 |
| `includeDeletedCount` | boolean | 默认 true，返回回收站数量 |

**Response**：
```json
{
  "code": 0,
  "data": {
    "todaySummary": {
      "totalOut": 12,
      "totalIn": 8,
      "diff": 4
    },
    "deletedCount": 2,
    "list": [
      {
        "id": "uuid",
        "name": "兴旺缝盘厂",
        "styleCount": 3,
        "role": "owner",
        "createdAt": "2026-06-01T10:00:00+08:00"
      }
    ]
  }
}
```

**权限**：返回当前用户作为 Owner 或 Member 的所有未删除工厂。  
**排序**：`created_at DESC`  
**todaySummary**：汇总用户所有工厂当日的发出/收回/差额。

---

### 6.2 获取单个工厂

```
GET /factories/{factoryId}
```

**权限**：`checkAccess(factoryId)` — Owner 或 Member。

---

### 6.3 创建工厂

```
POST /factories
```

**Request**：
```json
{
  "name": "兴旺缝盘厂"
}
```

**校验**：名称非空，trim 后长度 1–100。  
**权限**：任意登录用户。  
**副作用**：设置 `owner_id = 当前用户`，插入 `factory_members(role=owner)`。

---

### 6.4 更新工厂

```
PATCH /factories/{factoryId}
```

**Request**：
```json
{
  "name": "新华后整厂"
}
```

**权限**：`checkOwner(factoryId)`。

---

### 6.5 软删除工厂（移入回收站）

```
DELETE /factories/{factoryId}
```

**权限**：`checkOwner`  
**副作用**：
- `factories.deleted = true`
- 级联软删除该工厂下所有 `styles`
- 不删除 `records`（保留至彻底删除）

**Response**：
```json
{
  "code": 0,
  "message": "已移入回收站"
}
```

---

### 8.2 创建收发记录

```
POST /factories/{factoryId}/styles/{styleId}/records
```

**Request**：
```json
{
  "type": "out",
  "date": "2026-06-15",
  "items": {
    "红色": 36
  },
  "remark": "急单"
}
```

**校验**：
- `type` ∈ `out|in`
- `date` 合法日期
- `items` 中颜色必须属于款式 `colors`
- 数量为正整数
- 同一 `(styleId, type, date)` 不可重复（若已存在应走 PATCH）

**权限**：`checkOwner`

**Response**：
```json
{
```

## 十四、核心业务算法

### 14.1 统计汇总

```typescript
function aggregateRecords(records: Record[], unitPrice: number) {
  let totalOut = 0, totalIn = 0;
  const colorMap = new Map<string, { out: number; in: number }>();

  for (const r of records) {
    for (const [color, qty] of Object.entries(r.items)) {
      const stat = colorMap.get(color) ?? { out: 0, in: 0 };
      if (r.type === 'out') {
        totalOut += qty;
        stat.out += qty;
      } else {
        totalIn += qty;
        stat.in += qty;
      }
      colorMap.set(color, stat);
    }
  }

  const diff = totalOut - totalIn;
  const payable = round2(totalIn * unitPrice);

  return { totalOut, totalIn, diff, payable, colorMap };
}
```

### 14.2 应付加工费

```
应付加工费 = 累计收回件数 × 加工单价
```

保留 2 位小数，四舍五入。

### 14.3 数量解析

```typescript
function parseQuantity(input: string): number {
  input = input.trim();
  if (/^\d+$/.test(input)) return parseInt(input, 10);

  const daMatch = input.match(/^(\d+(?:\.\d+)?)打(\d*)$/);
  if (daMatch) {
    const da = parseFloat(daMatch[1]);
    const extra = daMatch[2] ? parseInt(daMatch[2], 10) : 0;
    return Math.round(da * 12) + extra;
  }

  const pureDa = input.match(/^(\d+(?:\.\d+)?)打$/);
  if (pureDa) return Math.round(parseFloat(pureDa[1]) * 12);

  throw new ValidationError('格式错误，如: 36 / 3打 / 1打2');
}
```

## 十五、索引与性能

### 15.1 关键索引汇总

| 表 | 索引 | 用途 |
|----|------|------|
| factory_members | `(user_id)` | 用户工厂列表 |
| factories | `(owner_id, deleted, created_at DESC)` | 首页列表 |
| styles | `(factory_id, deleted, created_at DESC)` | 款式列表 |
| records | `(style_id, date DESC)` | 收发页加载 |
| records | `(factory_id, date)` | 统计聚合 |
| records | UNIQUE `(style_id, type, date)` | 防重复行 |
| settlements | UNIQUE `(factory_id, date_from, date_to)` | 结算 upsert |
| invitations | `(code)` | 邀请码查询 |

### 15.2 性能建议

1. **收发页**：单款式记录一次全量返回；超过 1000 条考虑 cursor 分页 + 客户端增量合并。
2. **统计页**：按 `factory_id + date` 范围 SQL 聚合，避免拉全量 records 到应用层。
3. **今日汇总**：Redis 缓存，key=`today_summary:{userId}:{date}`，TTL=5min，写 record 时 invalidate。
4. **factoryIds 列表**：登录后缓存到 JWT claim 或 Redis，成员变更时刷新。
