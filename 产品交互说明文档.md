## 一、项目概述

### 1.1 项目定位

面向东莞大朗毛织缝盘/后整加工厂的**外协收发记账工具**。工厂老板将毛料外发给上游加工厂（缝盘厂、后整厂等），需要记录每次发出的数量和收回的数量，自动计算当日差额、应付加工费，管理多个合作工厂，查看产量统计台账。

### 1.2 目标用户

- **主要用户**: 工厂老板（数据录入 + 查看统计 + 成员管理）
- **次要用户**: 受邀员工（只读查看，不可修改数据）

### 1.3 核心业务流程

```
创建上游工厂 → 创建款式（设定颜色+单价）→ 每日收发录入（发出/收回数量）
    → 自动计算差额和加工费 → 按工厂/日期查看统计 → 生成对账卡片分享
```

### 1.4 技术背景

- 当前实现: 微信小程序原生开发（WXML + WXSS + JavaScript）
- 后端: 腾讯云 CloudBase（云开发），全部数据云端存储，无本地数据库
- 用户身份: 微信 OPENID 自动识别，无需注册/登录
- 原生 App 对接: 需自行实现登录系统（手机号/微信登录），后端需复刻所有云函数逻辑

## 二、整体架构

### 2.1 底部 3 栏 Tab 导航

App 底部固定 3 个 Tab，使用自定义 TabBar（非系统默认），在所有主页面始终可见：

| Tab | 图标 | 文字 | 对应页面 | 说明 |
|-----|------|------|----------|------|
| 首页 | 房子 SVG | **首页** | 工厂列表 | App 启动默认页 |
| 统计 | 柱状图 SVG | **统计** | 统计对账 | 数据汇总查看 |
| 我的 | 人像 SVG | **我的** | 个人中心 | 成员管理、回收站 |

- 选中态: 蓝色文字 + 蓝色图标（`#3B6FF5`）
- 未选中态: 灰色文字 + 灰色图标（`#8C8C8C`）
- 背景: 白色 `#FFFFFF`，顶部 1px 分割线
- 高度: 约 110rpx（≈55px），底部自动适配安全区

### 2.2 页面导航层级

除 3 个 Tab 页外，其余页面通过点击跳转进入，按返回键逐级返回：

```
首页（工厂列表）
  ├── 工厂编辑（新建/编辑）
  ├── 款式列表 → 款式编辑（新建/编辑）
  │             → 收发记账（核心录入页）
  └── 回收站

统计（统计对账）
  ├── 月度汇总（单款式明细）
  └── 对账卡片（分享卡片）

我的（个人中心）
  ├── 工厂回收站
  └── 款式回收站
```

### 2.3 全局通用功能

#### 2.3.1 登录校验

- App 启动时自动调用微信登录，获取用户 OPENID
- OPENID 是用户唯一标识，贯穿所有数据操作
- 登录失败不影响浏览，但所有数据操作无法执行
- 原生 App 需替换为自有登录体系（手机号/微信开放平台登录）

#### 2.3.2 全局加载弹窗

- 所有写操作（创建、更新、删除）自动弹出遮罩 loading："处理中…"
- 操作完成后自动消失
- 网络请求失败时消失并弹出错误 toast

#### 2.3.3 错误提示

- 网络异常: Toast "网络异常，请重试"（灰色失败图标）
- 操作失败: Toast 显示具体错误信息（灰色失败图标）
- 权限不足: "无权操作" / "无权查看该工厂数据"

#### 2.3.4 下拉刷新

以下页面支持下拉刷新（下拉后重新加载数据）:
- 首页（工厂列表）
- 款式列表
- 收发记账
- 统计对账
- 我的
- 回收站

## 三、全局通用交互规则

### 3.1 按钮规范

所有按钮高度统一为 **88rpx（≈44px）**，满足移动端最小触控面积要求。

| 按钮类型 | 样式 | 使用场景 |
|----------|------|----------|
| **主按钮**（Primary） | 蓝色实底 `#3B6FF5` + 白色文字 | 保存、创建、确定等正向操作 |
| **危险按钮**（Danger） | 红色实底 `#E54545` + 白色文字 | 删除确认 |
| **轮廓按钮**（Outline） | 透明底 + 蓝色边框 `#3B6FF5` + 蓝色文字 | 生成对账卡片等次要操作 |
| **危险轮廓按钮**（Danger Outline） | 透明底 + 红色边框 + 红色文字 | 编辑页的"删除此工厂/款式" |

**按压反馈**: 所有按钮按下时透明度降至 0.7-0.85，轻微缩放（97%-98%），过渡时间 0.15-0.2 秒。

### 3.2 触控规范

- 最小可点击区域: 88rpx × 88rpx（≈44×44px）
- 相邻可点击元素间距: 不小于 16rpx（≈8px）
- 表格行高: 88rpx
- 输入框最小高度: 88rpx（文本输入可为 72rpx）

### 3.3 弹窗规范

**所有弹窗遵循统一结构**:
- 背景: 半透明黑色遮罩（`rgba(0,0,0,0.5)`），覆盖全屏

## 四、页面 1：首页（工厂列表）

### 4.1 页面用途

App 启动后的默认首页。展示当前用户所有上游合作工厂，提供今日收发概览，支持搜索、新建、编辑、删除工厂，进入款式管理和收发记账。

### 4.2 页面 UI 布局

```
┌──────────────────────────────┐
│  🔍 搜索工厂                  │  ← 搜索栏
├──────────────────────────────┤
│  ┌──────┬──────┬──────┐      │
│  │ 今日 │ 今日 │      │      │  ← 今日汇总仪表盘
│  │ 发出 │ 收回 │ 差额 │      │     (始终显示)
│  │  12  │  8   │  +4  │      │
│  ├──────┴──────┴──────┤      │
│  └────────────────────┘      │
├──────────────────────────────┤
│  ┌──────────────────────┐    │
│  │ 兴旺缝盘厂    3款  ⋮ │    │  ← 工厂卡片（可滑动列表）
│  ├──────────────────────┤    │
│  │ 新华后整厂    1款  ⋮ │    │
│  ├──────────────────────┤    │
│  │ ...                   │    │
│  └──────────────────────┘    │
├──────────────────────────────┤
│  回收站（2个已删除工厂）  ›   │  ← 回收站入口（有删除项时才显示）
│  [  ＋ 添加上游工厂  ]       │  ← 固定底部按钮
└──────────────────────────────┘
```

**布局说明**:
- 搜索栏: 白色圆角输入框，占位文字"搜索工厂"，输入即过滤
- 今日仪表盘: 白色卡片，三栏等宽分布
  - 左栏"今日发出": 蓝字大数字（44rpx 加粗）+ 浅蓝渐变底
  - 中栏"今日收回": 绿字大数字 + 浅绿渐变底
  - 右栏"差额": 橙/红/绿色数字（根据正负自动变色）
  - 两栏之间用 2rpx 竖线分割
  - 无记录时差额显示"—"
- 工厂卡片: 白色圆角卡片（20rpx 圆角），左侧工厂名（34rpx 加粗）+ 款式数量蓝色徽章（如"3 款"），右侧"⋮"更多按钮
- 工厂卡片按压: 轻微缩放（98%）+ 阴影加深
- 空状态: 居中大"＋"图标 + "还没有上游工厂" + "点击下方按钮添加"（灰色提示）
- 底部按钮: 固定在 TabBar 上方，始终可见

### 4.3 按钮点击交互

| 操作 | 触发方式 | 交互响应 |
|------|----------|----------|
| **搜索工厂** | 输入文字 | 实时过滤工厂列表（按名称匹配，不区分大小写） |
| **进入款式管理** | 点击工厂卡片 | 跳转到款式列表页，传递工厂 ID 和名称 |
| **更多操作** | 点击"⋮"按钮 | 弹出系统操作菜单：[编辑] [删除（红色）] |
| **编辑工厂** | 菜单选"编辑" | 跳转到工厂编辑页，传递工厂 ID 和名称 |
| **删除工厂** | 菜单选"删除" | 弹出二次确认弹窗 → 确认后移入回收站 → Toast "已移入回收站" → 刷新列表 |
| **新建工厂** | 点击"+ 添加上游工厂" | 跳转到工厂编辑页（新建模式） |
| **回收站** | 点击回收站入口 | 跳转到回收站页（工厂类型） |
| **下拉刷新** | 顶部下拉 | 重新加载工厂列表和今日统计 |
| **长按工厂卡片** | 长按 | 同"⋮"按钮，弹出操作菜单 |

### 4.4 数据加载逻辑

1. 页面显示时（onShow）: 调用 `getMyFactoryIds()` 获取当前用户有权限的工厂 ID 列表
2. 如果工厂 ID 列表为空: 启动重试机制（最多 10 次，每次间隔 800ms），等待云函数返回
3. 用 ID 列表查询数据库 `factories` 集合（未删除的）
4. 同时查询所有工厂的款式数量（用于徽章显示）
5. 查询回收站数量（已删除工厂总数）
6. 查询今日收发统计（汇总所有工厂的当日记录）

**数据来源**: CloudBase 数据库直接查询（客户端 SDK），按 myFactoryIds 过滤
**排序规则**: 按创建时间倒序（最新的工厂在上）

### 4.5 弹窗交互

**删除确认弹窗**: 标题"警告"，内容"删除"{工厂名}"将同时删除其下所有款式和收发记录，确定？"，确定按钮红色文字"确定删除"，取消按钮默认样式。

### 4.6 页面跳转规则

| 目标页面 | 传递参数 | 参数说明 |
|----------|----------|----------|
| 款式列表 | `factoryId`, `factoryName` | 工厂唯一 ID + 名称（URL 编码） |
| 工厂编辑（新建） | 无 | 页面判断为新建模式 |
| 工厂编辑（编辑） | `factoryId`, `factoryName` | 工厂唯一 ID + 名称 |
| 工厂回收站 | `type=factory` | 回收站类型标识 |

## 五、页面 2：工厂编辑

### 5.1 页面用途

新建上游工厂或编辑已有工厂名称。页面根据是否传入 `factoryId` 自动判断新建/编辑模式。

### 5.2 页面 UI 布局

```
┌──────────────────────────────┐
│                              │
│  工厂名称                     │
│  ┌──────────────────────────┐│
│  │ 如：缝盘厂                ││  ← 输入框
│  └──────────────────────────┘│
│                              │
│  [ 保存 ]                    │  ← 蓝色主按钮
│                              │
│  [ 删除此工厂 ]  (仅编辑模式)  │  ← 红色轮廓按钮
│                              │
└──────────────────────────────┘
```

**布局说明**:
- 输入区: 白色圆角卡片
- 输入框: 88rpx 最小高度，1px 灰色边框，placeholder 提示"如：缝盘厂"
- 保存按钮: 蓝色实底，88rpx 高度
- 删除按钮: 仅在编辑模式下显示，红色轮廓，与保存按钮间距 40rpx

### 5.3 按钮点击交互

| 操作 | 交互响应 |
|------|----------|
| **保存** | 校验名称不为空 → 弹出 loading → 调用云函数创建/更新 → 成功 Toast + 自动返回上一页 |
| **删除** | 弹出确认弹窗 → 确认后移入回收站 → Toast → 返回上两级页面 |

**防重复提交**: 保存期间按钮不可再次点击（_saving 锁），防止创建重复工厂。

### 5.4 数据加载逻辑

- 新建模式: 不加载数据
- 编辑模式: 从数据库 `factories` 集合读取工厂名称，回填输入框

### 5.5 弹窗交互

**删除确认弹窗**: 标题"警告"，内容"删除工厂将同时删除其下所有款式和收发记录，确定继续？"，确定按钮红色。

### 5.6 页面跳转规则

- 保存成功 → 自动返回上一页
- 删除成功 → 自动返回上两级页面（因为款式列表页可能已不存在）

### 8.7 数据加载逻辑

#### 8.7.1 款式加载
1. 接收 `factoryId`、`styleId`、`styleCode` 参数
2. 校验 `factoryId` 是否在用户权限列表中
3. 查询该工厂所有未删除款式（分页查询，绕过 100 条限制）
4. 自动定位到传入的款式索引

#### 8.7.2 记录加载
1. 按当前选中的款式查询 `records` 集合
2. 一次拉取全部记录（分页查询），存入 `allRecords` 内存
3. 内存中按类型过滤（发出/收回）+ 计算统计（累计发出、累计收回）
4. 无记录时表格显示空行（列出各颜色按钮供点击录入）

#### 8.7.3 数据保存规则
- **新记录**（无 `_id`）: 调用云函数 `create` → 返回 `recordId`→ 内存更新（不重新拉取全部数据）
- **已有记录**（有 `_id`）: 调用云函数 `update` → 内存重新计算统计
- **日期修改**: 原有记录直接更新数据库；新记录仅内存更新
- **保存后**: 内存即时更新底部统计条数字，**不重新查询数据库**（避免闪烁和性能损耗）

#### 8.7.4 数据安全机制
- **竞态保护**: 快速切换款式时，旧请求返回结果被丢弃（_reqId 校验），确保显示数据与当前款式一致
- **防重复提交**: 保存期间 `_saving` 锁，快速连点不会创建多条重复记录

## 十五、数据流与权限

### 15.1 用户身份识别流程

```
App 启动 → wx.cloud.init() → 检查本地缓存 userInfo
  → 若有缓存: 直接使用 openid
  → 若无缓存: 调用 login 云函数 → 获取微信 OPENID → 缓存到本地 + 全局
  → loadMyFactoryIds() 云函数 → 获取用户有权限的工厂 ID 列表
```

原生 App 需自行实现: 手机号登录 → 后端生成唯一用户 ID → 替代 OPENID 体系。

### 15.2 数据库集合

| 集合名 | 存储内容 | 关键字段 |
|--------|----------|----------|
| `factories` | 上游工厂 | `name`, `ownerId`, `members[{openid,role}]`, `deleted`, `createdAt` |
| `styles` | 款式定义 | `factoryId`, `styleCode`, `unitPrice`, `colors[]`, `deleted`, `createdAt` |
| `records` | 收发记录 | `styleId`, `type(out/in)`, `date`, `items{color:qty}`, `remark`, `createdBy` |
| `settlements` | 结算标记 | `factoryId`, `dateFrom`, `dateTo`, `status`, `markedBy`, `markedAt` |
| `invitations` | 邀请码 | `factoryId`, `factoryName`, `inviterId`, `code`, `expireAt`, `createdAt` |

### 15.3 数据权限矩阵

| 操作 | 工厂 Owner | 工厂 Member | 其他人 |
|------|:--:|:--:|:--:|
| 查看工厂列表 | ✅ | ✅ | ❌ |
| 查看款式 | ✅ | ✅ | ❌ |
| 查看收发记录 | ✅ | ✅ | ❌ |
| 查看统计 | ✅ | ✅ | ❌ |
| 查看结算状态 | ✅ | ✅ | ❌ |
| 录入收发数量 | ✅ | ❌ | ❌ |
| 修改/删除记录 | ✅ | ❌ | ❌ |
| 创建/编辑/删除工厂 | ✅ | ❌ | ❌ |
| 创建/编辑/删除款式 | ✅ | ❌ | ❌ |
| 标记结算 | ✅ | ❌ | ❌ |
| 邀请成员 | ✅ | ❌ | ❌ |
| 移除成员 | ✅ | ❌ | ❌ |
| 恢复/彻底删除 | ✅ | ❌ | ❌ |

### 15.4 权限实现机制

- **写操作权限**: 所有数据写入（创建、更新、删除）通过云函数执行，云函数内通过 `checkOwner(factoryId, OPENID)` 校验 —— 从数据库读取工厂 ownerId 与调用者 OPENID 比对
- **读操作权限**: 通过 `getMyFactories` 云函数返回用户有权限的工厂 ID 列表（owner 或 member），客户端查询时按此列表过滤；云函数查询（stats、settlement）内同样校验 `checkAccess`（owner 或 member）
- **客户端防护**: 款式列表和收发记账页额外校验 URL 传入的 `factoryId` 是否在用户的工厂权限列表中，防止 URL 篡改

### 15.5 数据删除策略

- **软删除**: 工厂和款式删除时仅将 `deleted` 字段设为 `true`，数据保留在数据库中，可在回收站恢复
- **硬删除（彻底删除）**: 回收站的"彻底删除"会永久删除该记录 + 级联删除所有关联数据（工厂 → 款式 + 记录 + 结算 + 邀请；款式 → 记录）
- **记录删除**: 收发记录的删除为硬删除，直接从数据库移除
